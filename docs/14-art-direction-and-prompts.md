# Katha — Art Direction & Generation Prompt Pack

> The make-or-break visual layer. This doc holds (1) the locked art direction, (2) copy-paste
> **generation prompts** for every asset (Midjourney v7 / Flux), (3) the **character-consistency**
> strategy, and (4) the **drop-in asset spec** so generated images plug straight into the app.
>
> **How art is produced:** the procedural atmosphere (CSS/canvas/WebAudio in `app/web/index.html`)
> ships free and now; the painted hero assets below are generated by a human via Midjourney/Flux
> (or an illustrator) and dropped into `assets/` — see §4. The app uses real art when present and
> falls back to procedural otherwise.

---

## 1. Art direction (locked)

- **North star:** *Amar Chitra Katha* (reverent, iconic Indian illustration) × *Kalki 2898 AD*
  (cinematic, painterly, modern). Semi-realistic **painted**, NOT anime, NOT photoreal, NOT 3D.
- **Palette:** deep indigo/navy night base · amber/saffron key light · gold accents (#C9A84C).
  Per-scene tint below.
- **Mood:** moonlit, mythic, a little uncanny. Restraint over spectacle. Negative space for the
  dialogue panel (bottom ~40% of frame should be quieter/darker so text reads).
- **Composition for backgrounds:** 9:16 portrait (mobile-first), single strong focal element,
  atmospheric depth (fog/haze), low horizon, room for a character on the right third.
- **Reverence:** no deity is depicted irreverently (Phase 1 has none as principals); keep death/
  cremation-ground imagery solemn and atmospheric, never gory (13+).

### Per-scene tint & key elements
| Scene | Tint | Key elements |
|---|---|---|
| Prologue – Ujjain court (day) | warm gold/ochre | pillared hall, sunlight, a saffron-robed mendicant, a fruit/jewel |
| Cremation ground (Tales 1,2,5) | deep indigo + ember orange | lone śiṃśapā tree, low pyres, smoke, jackals, moon |
| River ghat (Tale 3) | teal/blue-green | stone steps to dark water, a cold pyre, jasmine, mist |
| Palace gate at midnight (Tale 4) | cold blue + torch amber | iron gate, torch, long shadows |
| Climax – the circle | blood-red + fire | a tantric circle, fire, the mendicant, a raised blade (implied, not gory) |

## 3. Character consistency (critical for portraits)

Generating 3 expressions that look like the *same* Betaal is the hard part:
- **Midjourney:** generate one strong base portrait, then use it as a **character reference**
  (`--cref <url> --cw 80`) for the other expressions; keep the same `--seed`.
- **Flux:** use a fixed seed + an **IP-Adapter / reference image**; or generate with a LoRA
  trained on 5–10 approved Betaal images for full control.
- Keep a `assets/characters/_reference/` folder with the approved base images and the seeds used,
  so the look is reproducible.
- **Advisor/taste gate:** pick the final Betaal look deliberately — it's the face of the product.

## 4. Drop-in asset spec (how generated art enters the app)

The web client (`app/web/index.html`) already looks for these paths and uses them if present,
else falls back to the procedural atmosphere. Served by `app/webserver.py` from `C:\Katha\assets\`.

```
assets/
├── backgrounds/<scene_id>.jpg        # e.g. cremation-ground-night.jpg  (matches scenes' `background`)
├── characters/<speaker>_<expr>.png   # e.g. betaal_amused.png (transparent)
│   └── <speaker>.png                 # expression-less fallback, e.g. kshantishila.png
└── ui/…                              # optional frames/ornaments
```
- **Background id** = the tale's `background` field in `app/content/tales.py`
  (`cremation-ground-night`, `ujjain-court-day`, `river-ghat-night`, `palace-gate-midnight`,
  `cremation-ground-circle-night`).
- **Portrait key** = lowercased speaker id + `_` + expression (`amused|intense|grave|neutral`),
  with `<speaker>.png` as the no-expression fallback.
- Recommended export: backgrounds 1080×1920 JPG; portraits ~800×1000 transparent PNG, optimized.

**That's it** — generate, name correctly, drop in `assets/`, reload. No code change.
